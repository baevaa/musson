Detail

The MUSSON system is designed to monitor the network,
in the ITU hierarchy, TMN implements the NMS+ umbrella management system with support for the functionality of the four levels of the FCAPS model

System functions

Alarms, malfunctions and events
  • Continuous 24/7 monitoring of equipment and services
  • Displaying an accident - with a counter of the number of operations and groupings
  • Full-featured filtering and searching
  • Masking a crash on certain hardware
  • Automatically requery a NE when there is a missing trap
Configuration, settings and composition of equipment
  • The composition of the equipment - for the analysis of the necessary spare parts and accessories, the possibility of movement
  • List of all installed licenses - to analyze the need to expand the functionality and upgrade the network element
  • Hardware configuration lists grouped by functionality
  • Data collection by schedule, by operator request, or by events on the network element
  • Control of the reliability of the collected data - algorithms for checking and reloading inform the system administrator about problems with loading
Equipment performance and degradation
  • Statistical analysis of network load and quality
  • Statistics of daily or interval parameter values
  • General list of bandwidth utilization % and Mbps - per hop, physical port and per VLAN
  • Creating a link congestion report - with the number of lost packets
  • Online real-time level measurements in the problem area
  • Quality of communication between the system (in the carrier cloud) and the equipment of the region
Security management
  • Management of network authentication, authorization
  • Audit

System specification

  • Support for SNMP(v1, v2c, v3), FTP, SSH, NetConf
  • Ability to collect data directly from NE
  • Download custom MIBs (from vendor) NEC Pasolink, Ericsson Minilink, Huawei RNT
  • Loading data structures, RMON data (RFC2819)
  • Grouping and masking accidents
  • Automatic NE Requery Due to Ignoring a Crash
  • Search for elements by network segment (Auto-Discovery)
  • Defining the network topology, displaying it on the network diagram
  • Georeferencing of objects on the map

Platform architecture

✖

The platform is built on the basis of modern open source software development systems, such as RabbitMQ, PostgreSQL, Timescale DB, Clickhouse.

The architecture includes the implementation of fault tolerance, clustering, horizontal and vertical scaling. The composition of functional blocks depends on the requirements for the system and compliance with the business logic of the operator's work.
